# 🤝 المساهمة في المشروع

شكراً لاهتمامك بالمساهمة في **منظم المذاكرة**! نرحب بمساهماتك سواء كانت:

- 🐛 تقارير عن الأخطاء
- ✨ ميزات جديدة
- 📝 تحسينات في التوثيق
- 🎨 تحسينات تصميم

## 📋 قبل البدء

1. تأكد من أن لديك حساب GitHub
2. اقرأ README.md لفهم المشروع
3. تحقق من [الـ Issues](../../issues) الموجودة

## 🚀 خطوات المساهمة

### 1. Fork المستودع
انقر على زر **Fork** في الأعلى اليمين

### 2. Clone نسختك
```bash
git clone https://github.com/YOUR_USERNAME/study-organizer.git
cd study-organizer
```

### 3. إنشء فرع جديد
```bash
git checkout -b fix/اسم-الميزة
```

أمثلة:
- `fix/timer-bug`
- `feature/add-notes-export`
- `docs/improve-readme`

### 4. قم بالتعديلات

### 5. اختبر التغييرات
- افتح الموقع في متصفح
- جرب جميع الميزات
- تأكد من عدم وجود أخطاء

### 6. Commit التغييرات
```bash
git add .
git commit -m "وصف واضح للتغيير"
```

#### قالب رسالة Commit:
```
[TYPE]: موضوع قصير (50 حرف أو أقل)

الوصف التفصيلي (اختياري):
- نقطة 1
- نقطة 2

يصلح #123 (رقم الـ Issue إن وجد)
```

أنواع التغييرات:
- `fix:` - إصلاح خطأ
- `feat:` - ميزة جديدة
- `docs:` - تحديث التوثيق
- `style:` - تصميم أو صيغة
- `refactor:` - إعادة هيكلة
- `perf:` - تحسين الأداء
- `test:` - إضافة اختبارات

### 7. Push للفرع الخاص بك
```bash
git push origin fix/اسم-الميزة
```

### 8. إنشء Pull Request
1. اذهب إلى المستودع الأصلي
2. انقر على **Pull requests**
3. انقر على **New pull request**
4. اختر فرعك ووصف التغييرات

## 📝 معايير الترمز

### HTML
- استخدم proper semantic HTML
- أضف تعليقات للأقسام الكبيرة
- استخدم data attributes للـ JavaScript

### CSS
- استخدم BEM naming convention
- استخدم CSS variables للألوان
- اختبر على عروض مختلفة

### JavaScript
- استخدم const و let (لا تستخدم var)
- اكتب functions صغيرة ومركزة
- أضف تعليقات للـ logic المعقد
- استخدم arrow functions

### Arabic Content
- استخدم كتابة عربية صحيحة
- تأكد من ترتيب الخوارزميات من اليمين لليسار
- اختبر على أجهزة RTL مختلفة
